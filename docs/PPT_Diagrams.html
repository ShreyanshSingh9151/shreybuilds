  • Lower memory footprint (~10MB vs ~200MB)
  • Faster cold start times

  • No email sending required
  • In-context AI assistance
  • Zero configuration for users

5. Methodology - Algorithm Details

🔍 Gmail DOM Extraction Strategy

// Multi-fallback selector approach for Gmail's dynamic DOM
const SELECTORS = {
    subject: ['h2[data-thread-perm-id]', 'h2.hP', 'div[role="main"] h2'],
    sender:  ['span[email].gD', 'h3 span[email]', 'span[email]'],
    body:    ['div.a3s.aiL', 'div.a3s', 'div[role="listitem"] div.a3s']
};

function extractVisibleThread() {
    const subject = trySelectors(SELECTORS.subject)?.textContent || 'No Subject';
    const sender = trySelectors(SELECTORS.sender)?.getAttribute('email') || 'Unknown';
    const bodies = document.querySelectorAll(SELECTORS.body[0]);
    const content = Array.from(bodies)
        .filter(el => isVisible(el))
        .map(el => el.innerText.trim())
        .join('\n\n-----\n\n')
        .substring(0, 12000);  // 12KB limit
    return { subject, sender, content };
}
Algorithm Characteristic Implementation Benefit
Stateless Processing Each request independent, no session state Horizontal scaling, no state sync
Provider Abstraction LLMProvider interface with Summarize(), Reply(), etc. Swap models without code changes
Graceful Fallback Mock providers when API keys unavailable Always functional demo mode
Cost Optimization Auto-route to cheapest effective model Gemini for classify = $0.001 vs $0.03

📊 Cost Estimation Formula

tokens = (len(input) + len(output)) / 4
cost = tokens / 1000 * rate[model]

Rates: GPT=$0.030, Claude=$0.015, Gemini=$0.001
